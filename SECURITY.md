# Política de seguridad

## Versiones soportadas

Snocomm Sentinel está en fase `0.1.0`. Hasta la publicación de una versión estable, las correcciones de seguridad se concentrarán en la rama `main` y en la versión más reciente disponible.

| Versión | Soporte |
| --- | --- |
| 0.1.x | Sí |
| < 0.1.0 | No |

## Reportar vulnerabilidades

Si encuentra una vulnerabilidad en Snocomm Sentinel, no la publique en un issue abierto si puede exponer a usuarios, organizaciones o evidencia sensible.

Reporte de forma privada mediante GitHub Security Advisories cuando el repositorio esté publicado. Si ese canal aún no está disponible, contacte a las personas mantenedoras del perfil Snocomm por el canal corporativo definido para seguridad.

Incluya, cuando sea posible:

- versión afectada;
- sistema operativo;
- comando ejecutado;
- impacto defensivo;
- pasos de reproducción no destructivos;
- evidencia mínima necesaria, redactada y sin datos personales;
- propuesta de mitigación.

## Alcance aceptado

Son bienvenidos reportes sobre:

- exposición accidental de datos sensibles en reportes;
- fallos de parsing que produzcan conclusiones engañosas;
- ejecución inesperada de comandos;
- escritura no prevista fuera del directorio de salida;
- errores de configuración que debiliten privacidad;
- vulnerabilidades de dependencias;
- bypasses de controles internos del propio toolkit.

## Fuera de alcance

No aceptamos ni recompensamos:

- explotación de terceros;
- pruebas contra sistemas sin autorización;
- payloads ofensivos;
- robo de credenciales;
- persistencia, evasión o bypass de antivirus;
- interacción con infraestructura real de atacantes;
- publicación de backups, datos personales o evidencia sensible.

## Compromiso de respuesta

El proyecto buscará confirmar recepción, evaluar impacto, proponer mitigación y publicar correcciones de forma proporcional. Al tratarse de un proyecto defensivo, se priorizarán fallos que puedan causar exposición de datos, conclusiones forenses incorrectas o uso inseguro en entornos corporativos.

## Divulgación responsable

Solicitamos un plazo razonable para corregir antes de publicar detalles técnicos. Cualquier divulgación debe minimizar daño y evitar instrucciones que puedan habilitar abuso.
