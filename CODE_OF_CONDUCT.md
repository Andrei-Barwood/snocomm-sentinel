# Código de conducta

## Propósito

Snocomm Sentinel es un proyecto open source defensivo. Su comunidad debe sostener un entorno profesional, respetuoso y orientado a reducir daño. La seguridad corporativa no justifica abuso, intimidación, vigilancia desproporcionada ni trato degradante hacia personas usuarias, mantenedoras o colaboradoras.

## Comportamientos esperados

- Participar con respeto, precisión técnica y buena fe.
- Distinguir hallazgos técnicos de conclusiones forenses.
- Usar lenguaje cuidadoso al tratar incidentes que puedan afectar a personas.
- Reportar vulnerabilidades por canales responsables.
- Respetar privacidad, consentimiento y límites legales.
- Aceptar correcciones editoriales, técnicas o éticas cuando mejoren el proyecto.

## Comportamientos no aceptados

- Acoso, discriminación, amenazas o ataques personales.
- Publicación de datos personales, credenciales, backups o evidencia sensible sin autorización.
- Instrucciones ofensivas, explotación de terceros, payloads, evasión, persistencia o robo de información.
- Uso del proyecto para vigilancia abusiva de empleados, clientes o terceros.
- Afirmaciones forenses sin evidencia verificable.
- Promesas absolutas de detección, eliminación o inmunidad frente a spyware mercenario.

## Alcance

Este código aplica a issues, pull requests, discusiones, documentación, eventos, canales de soporte y cualquier espacio asociado al proyecto.

## Aplicación

Las personas mantenedoras pueden moderar discusiones, editar o retirar contenido, cerrar issues, rechazar contribuciones o limitar participación cuando una conducta contradiga este código. Las decisiones deben ser proporcionales, documentadas cuando corresponda y orientadas a proteger la comunidad.

## Reportes

Para reportar una conducta inapropiada, use el canal definido en [SECURITY.md](SECURITY.md) si el caso involucra seguridad o privacidad. Para asuntos comunitarios no sensibles, abra un issue con información suficiente y sin exponer datos personales.
